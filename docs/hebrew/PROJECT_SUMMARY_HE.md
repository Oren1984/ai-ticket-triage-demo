# סיכום פרויקט – AI Ticket Triage + RAG Assistant (גרסת דמו)

## 🎯 מטרת המערכת

לבנות מערכת בינה מלאכותית מקצה־לקצה שמקבלת קריאת שירות (Ticket) בתחום IT / DevOps ומבצעת:

1. סיווג קטגוריה (למשל: Network, Database, Deployment וכו’)
2. קביעת רמת דחיפות (Low / Medium / High)
3. שליפת מסמך Runbook רלוונטי (RAG)
4. יצירת תשובה מסכמת לפתרון התקלה

המערכת בנויה כ-MVP רזה וללא תלות ב-GPU או שירותים חיצוניים.

---

## 🏗️ מה נבנה בפועל

### 📊 Dataset
- 1,002 קריאות שירות סינתטיות
- 6 קטגוריות מאוזנות
- 3 רמות דחיפות
- יצירה דטרמיניסטית לשחזור מלא

### 🤖 מודל סיווג (ML Baseline)
- TF-IDF + Logistic Regression
- מודל אחד לקטגוריה
- מודל אחד לדחיפות
- Category F1 = 1.00 (על דאטה סינתטי)
- Urgency F1 ≈ 0.29 (מוקצה אקראית בדאטה – צפוי)

### 📚 RAG
- 20 מסמכי Runbooks אמיתיים (Markdown)
- אחסון ב-Chroma Vector Store
- שליפה Top-3 צ’אנקים רלוונטיים
- Embeddings מדומים (ללא API חיצוני)

### 🌐 Backend
- FastAPI
- Endpoint: POST /predict
- לוגים מובנים (JSON)
- request_id ייחודי לכל בקשה
- מדידת latency

### 🧪 בדיקות
- 14 טסטים (Unit + Integration)
- כל הטסטים עוברים
- בדיקה מלאה של הזרימה: classify → retrieve → generate

### 🐳 Docker
- Dockerfile + docker-compose
- הרצה מלאה בפקודה אחת:
  docker compose up --build
