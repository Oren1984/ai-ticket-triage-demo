# דוח בילד – AI Ticket Triage + RAG Assistant (גרסת דמו)

## 🐳 מה קורה בזמן docker compose build?

במהלך הבילד מתבצעים שלושה שלבים מרכזיים:

1. 📦 התקנת תלויות (requirements.txt)
2. 🤖 אימון מודל הסיווג (TF-IDF + Logistic Regression)
3. 📚 אינג’סט של מסמכי Runbooks למערכת ה-RAG (Chroma)

כלומר — הקונטיינר נבנה כבר עם:
- מודלים מאומנים
- מאגר ידע מוכן לשליפה
- כל התלויות מותקנות

⏱ זמן בילד ראשוני: ~85 שניות  
⏱ בילד חוזר (ללא שינוי תלויות): ~6 שניות

---

## 🚀 מה קורה בזמן docker compose up?

1. הקונטיינר עולה
2. FastAPI נטען
3. השרת מתחיל להאזין לפורט 8000
4. אין צורך באימון או אינג’סט מחדש

⏱ זמן עלייה: פחות משנייה  
המערכת מוכנה מיידית לקבל בקשות.

---

## 🎯 למה בחרנו לבצע אימון בזמן Build?

✔ הקונטיינר עולה מיד  
✔ אין עיכוב בזמן הדמו  
✔ אם האימון נכשל — הבילד נכשל (אין מצב למערכת שבורה)  
✔ תוצאה דטרמיניסטית וניתנת לשחזור  

---

## ⚠️ תרחישי כשל אפשריים

- חוסר בתלות ב-requirements.txt → הבילד ייכשל
- data חסר → האימון ייכשל
- knowledge_base ריק → RAG לא יחזיר תוצאות
- פורט 8000 תפוס → הקונטיינר לא יעלה

בכל המקרים — הכשל מתגלה בשלב הבילד או העלייה.
